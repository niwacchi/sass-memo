# Sassとは
## 基本
- 公式サイト：https://sass-lang.com
- CSSの機能を拡張したメタ言語
- Syntactically Awesome Stylesheets の略
- コードの再利用、変数、演算、条件分岐の機能をもたせた
- CSSとの互換性がある
## sassとscss の違い
### SASS記法
- Sass記法
-  {}（波括弧）の代わりにインデントで表現する。
-  ;（セミコロン）を省略できる。
-  簡素化された記法
-  CSSの記法との互換性が無い
### SCSS記法
- Sassy CSS の略
- CSSと書式が同じ
## 使い方
- CSSにコンパイルする必要がある。
- Ruby ＋ Sass
- Node.js ＋ noda-sass

## Sassのメリット
- 記述の簡略化
- 「// 〜」コメントも使える
- 同じ値を変数に定義して使い回すことができる。
- セレクタを使いまわすことができる。(extend）
- コード再利用ができる。（mixin）
- １つのファイルにまとめて、リクエスト回数をへらすことができる。（パーシャル）
- 条件分岐ができる。
- 関数を使うことができる。
- CSSファイルの圧縮・軽量化ができる。（コンパイル時）
